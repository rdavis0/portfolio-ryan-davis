Objects

  • All objects are mutable while program is running
  • Use 'property' in object to check if object has a property. hasOwnProperty checks for non-inherited properties
  • Objects are passed by reference. Copies point to the same object in memory. The opposite is true of primitives.
  • An object can be passed as a parameter to a function, allowing you to pass the parameters in any order

The Document Object Model

  • Avoid having to insert nodes in the tree with innerHTML

Events

  • Event listeners are a non-blocking approach to handling events
  • click even counts mouse clicks, enter key, and touchscreen taps
  • Event listeners are the current standard. document.body.addEventListener('click', myFunction);
  • Touch events are used for smartphones and tablets
  • There are no swipe events; must be built with a combination of touchstart, touchmove, and touchleave, or use a library
  • Event propagation
    • Bubbling (default) is when an event fires on the element clicked first, then bubbles up the document (e.g. li to ul to body to html)
    • Capturing is when the the event fires on the root element first, then propagates downwards through children to the target element (e.g. ul to li)
