Forms

  • Get collection of forms from a page with document.forms
  • Forms trigger a number of Events
    • focus
    • blur when an element loses focus
    • change when an element loses focus after being changed
    • submit
  • Use value attribute to read content of form fields
  • placeholder attribute prefills the field with helper text that is not submitted if the user doesn't replace it
  • event.preventDefault() can be used in a script to prevent the form from being submitted
  • Forms can contain various input field types
    • text (default)
    • password
    • checkbox; use the same name property to access them as an HTML collection like form.myName
    • radio
  • Forms fields can be hidden with type='hidden'
  • type='file' to upload files in a form field. Here's the default browser implementation:
  • Other input types like number, tel, color can be used to trigger different virtual keyboards or input elements. Examples below:

  • Form validation can be done via HTML5 or custom Javascript. The text says HTML5 is not widely supported, but I don't think that's still the case.

Object Oriented Programming in Javascript

  • Javascript started as a prototype-based language, meaning an actual object is used as the blueprint for creating more objects (as opposed to class-based)
  • JavaScript added class support in ES6
  • Two ways to create a new object:
    • const dice = {
    sides: 6,
    roll() {
    return Math.floor(this.sides * Math.random() + 1)
    }
    }
    const Dice = function(sides=6){ this.sides = sides; this.roll = function() { return Math.floor(this.sides * Math.random() + 1) } }
  • Instantiate the object with const d20 = new Dice(20);
  • What is the purpose of built-in Object constructor? obj = new Object();
  • Recommended to use literals to construct arrays const arr = [1, 2, 3] because of strange behavior when using object constructor Array()
  • Use static keyword in class declaration for methods called by the class directly rather than instances of the class
  • The prototype property can be used to add new default properties or methods after the declaration: Turtle.prototype.weapon = 'Hands';
