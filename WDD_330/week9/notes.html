Week 9

The Window Object

  • The window object is the global object; contains global variables and methods as its properties
  • this in the global scope refers to the global object
  • alert(), confirm(), and prompt() are methods of the window object
  • The Navigator object contains information about the browser. Access with window.navigator
  • window.location contains information about the URL of the current page
  • Use window.history to access info about history in the current session

Cookies

  • Create a cookie with document.cookie = 'foo=bar';
  • Cookies are read with document.cookie, but come back as a single string:
    document.cookie:
    << "name=Batman; hero=true; city=Gotham"
  • Cookies expire at the end of the browser session but can be made to last longer with expires=date or max-age= //value in seconds
  • Libraries like Cookies.js or jsCookie make cookie management easier

HTML5 APIs

data- property

  • data- attribute embeds data with an element that is ignored by the browser
  • Access an element's data- properties with element.dataset.myProperty

Web Storage API

  • Web Storage API (localStorage) lets you save key-value pairs. Lots of advantages over cookies.
  • localStorage.name = 'Falco';
  • localStorage.removeItem('name'); or delete localStorage.name
  • localStorage.clear() removes everything stored in local storage (for that domain)

Web Workers

  • Web workers run processes in the background, allowing for concurrency
  • const worker = new Worker('someFile.js');
  • postMessage() will send a message and start the worker working
  • From the main file: worker.postMessage('Hello')
  • Within the worker file self.postMessage('Finished')
  • A message event is fired when a message is posted. Can be used in callbacks.

Web Sockets

  • Web sockets allow for an open line of communication between the server and the client, so each side can push updates to the other; think messaging application
  • They use secure protocol wss://
